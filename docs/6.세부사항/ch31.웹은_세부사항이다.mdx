# 31장 웹은 세부사항이다

## 31-1. 끝없이 반복하는 추

- 연산 능력을 중앙에 집중하는 방식과 분산하는 방식 사이에서 끊임없이 움직인다.
- 웹은 아무것도 바꾸지 않았고, 수많은 진동 중 하나에 불과하다.

## 31-2. 요약

- GUI는 세부사항이다. 웹은 GUI다.
- 아키텍트라면 이러한 세부사항을 핵심 업무 로직에서 분리된 경계 바깥에 두어야 한다.
- UI와 애플리케이션 사이에는 추상화가 가능한 또 다른 경계가 존재한다.
- 업무 로직은 다수의 유스케이스로 구성되며, 각 유스케이스는 사용자를 대신해 일부 함수를 수행하는 것.
- 각 유스케이스는 입력 데이터, 수행할 처리 과정, 출력 데이터를 기반으로 기술할 수 있다.
- UI와 애플리케이션이 함께 돌아가는 동안, 어떤 시점이 되면 입력 데이터가 완전히 구성되고, 해당 유스케이스를 실행할 수 있게 된다.
- 유스케이스가 종료되면 해당 입력 데이터에 따른 결과 데이터는 UI와 애플리케이션이 함께 움직이게 되돌려줄 수 있다.
- 완전한 입력 데이터와 그에 따른 출려 데이터는 데이터 구조로 만들어 유스케이스를 실행하는 처리 과정의 입출력 값으로 사용할 수 있다.
- 각 유스케이스가 장치 독립적인 방식으로 UI라는 입출력 장치를 동작시킨다고 간주할 수 있다.
